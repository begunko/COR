# 🛤️ ROADMAP — COR Engine

> **Статус:** Pre-Alpha (v0.2.0)  
> **Последнее обновление:** 2026-05-12  
> **Архитектура:** Django + Daphne (ASGI) + Babylon.js + WebSocket

---

## 🎯 Видение

COR Engine — это браузерный 3D-движок для **совместного** редактирования игровых миров с шестигранными чанками (сотами).  
Цель: дать командам (художникам, дизайнерам, программистам, маркетологам) гибкий инструмент, который растёт вместе с пользователем — от новичка до профи.

**Принципы:**
- **Open Source:** всё в открытом доступе, формат документации — Markdown.
- **Ролевая модель:** интерфейс адаптируется под права (Маркетолог не видит гизмо).
- **Чанки-соты:** мир делится на шестигранные призмы. Радиус чанка — 50м.
- **Совместность в реальном времени:** WebSocket-комнаты, soft-lock объектов.
- **Гибкие настройки:** пользователь сам решает, создавать ли объекты с коллизией по умолчанию.

---

## 📦 Текущая архитектура (Что уже есть)

- **Django-приложения:**
  - `assets` — составные объекты (JSON-описание геометрии).
  - `projects` — контейнеры верхнего уровня (одна игра = один проект).
  - `scenes` — миры (`World`), чанки (`Chunk`), объекты (`WorldObject`).
  - `tools` — реестр инструментов (`Tool`), наборы (`Toolkit`), категории (`Category`).
  - `users` — кастомная модель `User`, права `ProjectPermission`.
- **ASGI:** Daphne на порту 8000 (WebSocket), Django runserver на 8001 (статика + API).
- **Фронтенд:** Babylon.js (модули `BabylonScene`, `ObjectManager`, `RealtimeSync`, `Inspector`, `BabylonToolbar`).
- **Чанки:** модель `Chunk` с гексагональной математикой. 18 соседей (6 горизонтальных + 6 верхних + 6 нижних).

## 🧩 Фазы развития

### Фаза 1: Стабилизация и базовые фичи (v0.3.0)
**Цель:** Сделать редактор удобным для одного дизайнера.

- [ ] **1.1 Инспектор с редактированием**
  - Сейчас: только отображает позицию/поворот/масштаб.
  - Надо: добавить поля ввода (`<input>`) в GUI-панель.
  - Реализовать арифметику в полях (`10+5/2`).
  - Связать поля с `ObjectManager.updateObject()` и отправкой через `RealtimeSync.sendUpdate()`.

- [ ] **1.2 Soft Lock объектов**
  - Сейчас: `ChunkConsumer` рассылает `object_updated` всем.
  - Надо: при начале редактирования отправлять `lock_object`, при потере фокуса — `unlock_object`.
  - В `Inspector` блокировать поля, если получен `lock` от другого пользователя (серый цвет, иконка замка).

- [ ] **1.3 Prefab Defaults (настройки создания)**
  - Сейчас: `Tool.default_params` задаёт геометрию, но не коллизию/физику.
  - Надо: в `User.tools_config` добавить поле `default_collision: true/false`.
  - При создании объекта через `BabylonToolbar` проверять эту настройку и включать `mesh.checkCollisions`.

- [ ] **1.4 Snap to Surface (магнит к земле)**
  - Сейчас: объект создаётся на фиксированной высоте.
  - Надо: при перетаскивании делать рейкаст вниз, устанавливать `position.y` на поверхность.
  - Кнопка-переключатель на панели гизмо.

- [ ] **1.5 Performance Profiler (окно FPS)**
  - Сейчас: нет.
  - Надо: отдельная GUI-панель (как Inspector), показывает текущий FPS, количество draw calls, загруженные чанки.
  - В Outliner подсвечивать "тяжёлые" объекты красным.

### Фаза 2: Совместная работа и чанки (v0.4.0)
**Цель:** Настоящий мультиплеер и бесшовный мир.

- [ ] **2.1 Сфера загрузки чанков (19 сот)**
  - Сейчас: `BabylonScene` одна статичная, `Chunk.load_chunk()` возвращает объекты одного чанка.
  - Надо: при движении камеры определять центральный чанк, запрашивать через API 19 чанков (центральный + 6 вокруг + 6 сверху + 6 снизу).
  - Рендерить объекты всех загруженных чанков.

- [ ] **2.2 Предзагрузка (LOD)**
  - Чанки на расстоянии > 1 слоя загружаются с пониженной детализацией (меньше полигонов, нет теней).
  - Использовать `mesh.simplify()` или заранее сгенерированные LOD-модели.

- [ ] **2.3 Outliner (дерево сцены)**
  - Сейчас: только HTML-список в `entity_editor.html`.
  - Надо: отдельная GUI-панель с вкладками:
    - **World Map:** иерархия `Чанк → Группа → Объект`.
    - **Prefabs:** список доступных Tool/Asset для перетаскивания в Viewport.
    - **Data Graph:** зависимости ассетов (кто ссылается).

- [ ] **2.4 Чат и голосовая связь**
  - Сейчас: нет.
  - Надо: текстовый чат через WebSocket (та же комната чанка).
  - Голос: WebRTC (опционально, SFU сервер).
